accessibility testing

  • overview

    The ‘Route Page’ of the trip planner embedded in the travel website ‘VisitFinland’ is the first page users encounter when they start planning their trip. This page points them in the direction of useful links and information so that they customize their trip according to their tastes. It is a very crucial part of their experience with the entire system and therefore, I specifically chose to evaluate the accessibility of this particular web page.

    client: Graduate Course
    when: Dec 2021
    role: UX Researcher
    methods: Manual Accessibility Inspection, Automated Accessibilty Evaluation Tool
    tools: Google Chrome Version 96.0.4664.93, VoiceOver Version 10 (804), WAVE Chrome Extension, Figma


  • does the route page meet the user's accessibility needs?

    To answer this question, I performed manual and automated accessibility evaluations. For the manual evaluation, I employed the Web Content Accessibility Guidelines (WCAG) 2.1, which are based on the four principles: Perceivable, Operable, Understandable and Robust. Following through, I further evaluated the accessibility using WAVE's Chrome Extension. This tool directly evaluated the accessibility right in the browser.


  • top accessibility issues
    missing label for search input field

    The search bar at the top of the page does not have a label and therefore is inaccessible. This is an important feature and adds to the usability of the system tremendously. It allows users to look for activities they might be interested in and places they would like to explore. It is the starting point for users that want to customize their itinerary.


    dropdown content not accessible

    The ‘Day by day’ element on the trip navigation bar has a dropdown menu that is visible when users hover over the down arrow. The ‘Day by day’ views are robust and they display important information about the trip. It also has options to allow users to customize their trips. They offer a high-level overview of the trip in a way that is easy to understand. As this dropdown is currently inaccessible, the users are deprived of valuable information.


    link purpose not identified in card main content

    Each of the above elements is a link but they are not accessible currently. These links are extremely useful when the user is just beginning to explore their itinerary. It redirects them to fun activities that could make their trip very memorable. This improvement has the potential to drastically improve the user experience.

  • retrospective
    most links on route page are inaccessible

    The Route page meets most accessibility standards except for the links, most of which are either inaccessible or fail to convey their purpose. Apart from links, a couple of images lack alt-text and a couple of links are empty.

    manual inspection vs. automated scan

    The automated testing works best in scenarios where you’re looking to double-check things or when you have content that is a mix of fixed components. However, in cases where the code is alien and the tester has no knowledge of whether the correct HTML tags are used; manual inspection can find such deeper issues that can significantly improve accessibility.
