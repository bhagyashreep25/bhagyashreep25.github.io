amazon alexa - usability testing

  • overview

    Alexa is a virtual voice assistant owned by Amazon. It has diverse features to offer such as music playback, to-do lists, setting alarms, providing weather, creating a shopping list, etc. VUIs have started helping the user off-load their memory and automate basic tasks by giving voice instructions. The challenge with VUI is that, unlike graphical user interfaces that set visual and mechanical constraints for user interactions, the voice interface does not have a machine-friendly framework under which the interactions would happen. Rather the framework is driven by the rules of language and we are training the machine to read natural language. With the Alexa mobile app (VUI + app UI), I wanted to evaluate how the user navigates between the voice assistant and the mobile app itself. Investigate how both interactive UIs work together or have discrepancies that confuse the user.

    client: Graduate Course
    when: Feb-May 2022
    role: UX Researcher on a team of three
    Process: Heuristic Review, Persona Development, Task Analysis, Paths & Scenarios, Test Plan, Usability Testing, Analysis
    tools: Figma, Google Suite


  • heuristic review


  • persona

    I created a persona that depicts a typical archetype of the average user. It allowed me to think of how Alexa would be used and how it fits into the life of the primary user. It also served as an opportunity to do research on Alexa's target audience and better conduct the testing.




  • task analysis

    I defined a set of tasks and categorized them further into sub-tasks by performing an analysis of the various skills offered by Alexa. I then dived deep into each of them to find out the common sub-tasks or individual commands that triggered a basic action. This also indicated what potential tasks could be included in the test plan in consideration of the persona.


  • path analysis

    I identified and listed the multiple ways to complete the tasks identified previously. I then created scenarios to determine the flow of tasks and how each of the different skills would interleave for the duration of the test. This allowed me to progress from research to execution of the usability test.


  • test plan

    I combined the list of tasks using scenarios that interleave them into a test plan. I added some ice breaker questions, follow up questions and concluding questions to develop a framework that can be used while testing. It acted as the main reference document to be referred to while planning the logistics of the test. This plan also gave an opportunity to execute a dry run of the test and refine it. That ensured that the flow is smooth and the test is within the desired time constraints.
